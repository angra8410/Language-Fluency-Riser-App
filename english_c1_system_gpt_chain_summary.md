# English C1 System – Antoine  
## GPT Chain Summary

This document summarizes the full GPT chain for the **English C1 System – Antoine** project.  
Goal: move from **B2 to C1** in 24 weeks with a modular, adaptive, self-improving workflow.

---

# 1. System Goal

Build real-world English fluency with focus on:

- speaking
- writing
- listening
- reading
- vocabulary depth
- grammatical accuracy
- discourse control
- reaction under pressure

Main principles:

- active recall
- deliberate practice
- progressive immersion
- retrieval before review
- error-based adaptation
- real-world task transfer

The system uses these GPTs:

1. **Supervisor GPT**
2. **Diagnostic Coach GPT**
3. **Speaking Lab GPT**
4. **Writing Studio GPT**
5. **Input Immersion GPT**

# 3. Role of Each GPT

## 3.1 Supervisor GPT
**Function:** orchestrator and memory manager.

### Responsibilities
- reads learner profile, session logs, progress tracker, review queue
- sets daily priorities
- chooses the next GPT order
- adjusts difficulty
- updates memory after sessions
- decides next-session focus
- monitors recurring weaknesses
- adapts the system over time

### Use it:
- at the **start** of every session
- at the **end** of every session
- for weekly adjustment
- after every 4-week checkpoint

---

## 3.2 Diagnostic Coach GPT
**Function:** evaluator.

### Responsibilities
- estimates CEFR level
- runs baseline assessments
- runs 4-week checkpoints
- scores speaking, writing, listening, reading
- compares performance against B2/C1 descriptors
- identifies strengths and weaknesses

### Use it:
- at the beginning of the program
- every 4 weeks
- when you want a formal progress check

---

## 3.3 Speaking Lab GPT
**Function:** speaking practice engine.

### Responsibilities
- builds fluency under pressure
- trains reaction speed
- improves natural phrasing
- develops debate, interruption handling, spontaneity
- pushes active speaking output
- provides limited, high-impact corrections
- forces self-repair and repetition

### Use it:
- on speaking days
- when fluency or reactivity is a priority
- for debate simulation
- for interruption training
- for timed speaking

---

## 3.4 Writing Studio GPT
**Function:** writing development engine.

### Responsibilities
- improves accuracy
- improves complexity and coherence
- trains essays, reports, emails, proposals
- reduces Spanish-to-English translation patterns
- develops natural phrasing and C1 writing control
- forces rewrite after feedback

### Use it:
- on writing days
- when grammar control needs work
- when written complexity is lagging
- when you need task-based writing practice

---

## 3.5 Input Immersion GPT
**Function:** reading/listening + output bridge.

### Responsibilities
- runs reading or listening practice
- trains gist, detail, inference, tone, structure
- extracts reusable chunks and collocations
- turns passive input into active output
- bridges texts/audio into speaking or writing

### Use it:
- after main production blocks
- on input-focused days
- when vocabulary in context is needed
- when listening/reading scores lag behind speaking/writing

---

## 3.6 Error Analyst GPT
**Function:** root-cause analysis.

### Responsibilities
- detects recurring patterns
- separates slips from stable weaknesses
- identifies likely causes
- prioritizes the top high-impact issues
- generates micro-drills
- creates transfer tasks
- suggests memory updates for Supervisor GPT

### Use it:
- after every main Speaking or Writing session
- after integrated tasks
- whenever you want sharper corrective strategy

---

## 3.7 Recall & Drill GPT
**Function:** consolidation engine.

### Responsibilities
- active recall
- spaced repetition
- short retrieval rounds
- grammar and chunk drills
- anti-repetition reformulation
- retrieval before production
- review scheduling

### Use it:
- **every day**
- before Speaking or Writing
- after recurring errors appear
- for review queue items

---

# 4. Daily Workflow

This is the standard daily chain.

## Standard order
1. **Supervisor GPT**
2. **Recall & Drill GPT**
3. **Main production GPT**
   - Speaking Lab GPT **or**
   - Writing Studio GPT
4. **Input Immersion GPT** (recommended)
5. **Error Analyst GPT**
6. **Supervisor GPT**
7. Save memory files in the Project

---

# 5. What Each Day Looks Like

## Step 1 — Supervisor GPT
Purpose:
- read memory
- decide what today’s priority is
- prescribe GPT order
- set difficulty and drills

### Typical question
“Based on my latest learner profile, progress tracker, review queue, and last session log, prescribe today’s session.”

---

## Step 2 — Recall & Drill GPT
Purpose:
- warm up retrieval
- reactivate yesterday’s errors
- strengthen automaticity
- prepare for production

### Typical duration
10–15 minutes

---

## Step 3 — Main Production Block
Choose one:

### Speaking day
Use **Speaking Lab GPT**

Focus examples:
- fluency
- interruptions
- debate
- timed speaking
- natural phrasing
- self-repair

### Writing day
Use **Writing Studio GPT**

Focus examples:
- essay or report writing
- grammar control
- lexical precision
- hedging
- paragraph logic
- rewrite after feedback

### Typical duration
20–35 minutes

---

## Step 4 — Input Immersion GPT
Purpose:
- feed your production with better English
- practice reading/listening
- extract chunks to reuse
- bridge comprehension into output

### Typical duration
10–20 minutes

Optional on short days, but strongly recommended on normal days.

---

## Step 5 — Error Analyst GPT
Purpose:
- find recurring patterns
- identify root causes
- rank the top 3 priorities
- generate corrective drills and transfer tasks

### Typical duration
5–10 minutes

---

## Step 6 — Supervisor GPT (Close the loop)
Purpose:
- update learner profile
- update progress tracker
- update review queue
- create today’s session log
- prescribe next session

This is the step that keeps the system adaptive.

# 11. Rule for Choosing the Main GPT

## Use Speaking Lab GPT when:
- fluency is the priority
- you need faster reaction time
- you want debate or pressure training
- spontaneity is weak
- you need natural spoken phrasing

## Use Writing Studio GPT when:
- written accuracy needs work
- grammar control is unstable
- lexical precision is weak
- your speaking is good but structure is still too B2 in writing
- you need essays, reports, or emails

## Use Input Immersion GPT when:
- you need more vocabulary in context
- you want better chunk acquisition
- listening or reading is weaker than speaking
- you need stronger inference or summarization ability

## Use Recall & Drill GPT:
- every day

## Use Error Analyst GPT:
- after every main production session

## Use Supervisor GPT:
- always at the beginning and end

---

# 12. Adaptive Logic

The system adjusts itself using these rules:

- if a weakness appears **3+ times**, increase training frequency
- if scores stay **above 85**, increase difficulty
- if scores fall **below 60**, simplify and reinforce fundamentals
- if fluency is weaker than accuracy, add more speaking pressure
- if accuracy is weaker than fluency, add more controlled writing and correction
- if complexity is weak, add lexical range, contrast, hedging, and argumentation drills
